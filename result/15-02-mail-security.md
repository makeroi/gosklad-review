# 15.2. Безопасность писем

## Результат
- **Статус:** ⚠️ Требует проверки
- **Severity:** S2
- **Краткое заключение:** не подтверждено наличие записей DKIM/SPF/DMARC и мониторинга доставляемости.

## Наблюдение
- В проекте нет описания настройки DNS-записей для домена отправки.
- Не зафиксированы показатели bounce/spam.

## Доказательства
- Отсутствует документация в репозитории
- Со слов команды: «почта работает, но контроля нет»

## Риски
- Письма могут попадать в спам, снижая эффективность уведомлений.
- Блокировка домена почтовыми провайдерами.

## Рекомендации
1. Проверить и настроить записи SPF, DKIM, DMARC.
2. Включить Postmaster/DMARC-отчёты и мониторинг bounce-rate.
3. Документировать процесс ротации ключей для DKIM.

- 📌 Контроль: ежемесячный отчёт по доставляемости.
