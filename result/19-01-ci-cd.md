# 19.1. CI/CD

## Результат
- **Статус:** ❌ Не пройдено
- **Severity:** S1
- **Краткое заключение:** деплой выполняется вручную по SSH; отсутствуют пайплайны с тестами, линтерами и прогревом кэшей.

## Наблюдение
- В репозитории нет `.github/workflows`, `.gitlab-ci.yml` или других конфигураций CI/CD.
- На звонке подтверждено: «деплоя нет», «работаем руками».

## Доказательства
- Отсутствие CI-конфигов
- Заметки со звонка

## Риски
- Высокий риск человеческих ошибок при релизах.
- Невозможность отката и контроля качества.
- Нет автоматического запуска тестов/линтеров.

## Рекомендации
1. Настроить CI (GitHub Actions/GitLab CI) с шагами: install → lint → test → build → deploy.
2. Реализовать zero-downtime деплой (Envoyer, Deployer, GitHub Actions + rsync).
3. Документировать процесс и предусмотреть автоматический откат.

## Следующие шаги
- 🟥 Ответственный: DevOps
- ⏱ Срок: 3 недели
- 📌 Контроль: каждый merge в main проходит через CI/CD без ручных шагов.
