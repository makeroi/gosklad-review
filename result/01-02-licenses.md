# 1.2. Лицензии и правовые риски

## Результат
- **Статус:** ⚠️ Требует проверки
- **Severity:** S2
- **Краткое заключение:** лицензии зависимостей не проанализированы; потенциальные риски не оценены.

## Наблюдение
На момент аудита не запускались `composer licenses` и `npm ls --json --long`. В репозитории отсутствует политика выбора зависимостей.

## Доказательства
- Нет отчёта по лицензиям среди артефактов аудита.
- `composer.json` и `package.json` — большое количество внешних пакетов без комментариев по лицензиям.

## Риски
- Возможное нарушение условий использования пакетов (GPL/LGPL) в коммерческом проекте.
- Финансовые и юридические последствия при передаче продукта заказчику.

## Рекомендации
1. Запустить `composer licenses --format=json` и `npm ls --json --long` (если используется фронтенд).
2. Составить перечень зависимостей с лицензиями и согласовать с юристами.
3. Документировать правила выбора пакетов (разрешённые/запрещённые лицензии).
