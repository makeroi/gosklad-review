# 11.1. Кэш-стратегии

## Результат
- **Статус:** ⚠️ Требует проверки
- **Severity:** S2
- **Краткое заключение:** политика кэширования не задокументирована; используется точечный кеш (Redis) без централизованной стратегии.

## Наблюдение
- Кэширование реализовано в отдельных методах (например, `Cache::remember('values', ...)` в `TestController`).
- Не описаны TTL, теги, политика инвалидации.
- Неизвестно, используется ли Redis в продакшне для кэша или только для очередей.

## Доказательства
- `app/Http/Controllers/Api/V1/TestController.php:48-50`
- Отсутствие описания политики в документации

## Риски
- Кэш может возвращать устаревшие данные (особенно при изменении остатков/резервов).
- Невозможно очистить кэш выборочно.

## Рекомендации
1. Определить список сущностей, которые необходимо кешировать (справочники, настройки, отчёты).
2. Использовать теги (`Cache::tags`) и TTL.
3. Документировать стратегию инвалидации (по событиям/по времени).
