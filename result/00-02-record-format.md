# 0.2. Формат фиксации находок

## Результат
- **Статус:** ✅ Соответствует чеклисту
- **Severity:** S3 (методологический пункт)
- **Краткое заключение:** структура описания проблем используется во всех файлах `audit/result/*`.

## Наблюдение
Формат «Наблюдение → Доказательства → Риск → Рекомендация → Срок → Владелец» помогает быстро понимать суть проблемы и план действий.

## Доказательства
- Все файлы в `audit/result/` оформлены по единому шаблону.
- `audit/result/SUMMARY.md` содержит таблицу с полями из формата.

## Риски
При отсутствии стандартизированной формы записи возрастает риск потери контекста, дублирования работ и недопонимания между командами. Сейчас риск минимален.

## Рекомендации
- Сохранить формат как обязательный для будущих аудитов.
- Дополнительно фиксировать дату закрытия задачи и ссылку на PR.
