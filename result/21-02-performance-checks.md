# 21.2. Как проверить

## Результат
- **Статус:** ⚠️ План требуется
- **Severity:** S2
- **Краткое заключение:** отсутствует формализованный процесс измерения производительности; необходимо внедрить план проверок.

## Наблюдение
- Нет профилировщиков/трасс (Blackfire, Telescope) в продакшне.
- Не настроено регулярное снятие slow query log.

## Доказательства
- Отсутствие упоминаний в документации и инфраструктуре

## Риски
- Улучшения производительности рыхлы и не проверяются после релизов.

## Рекомендации
1. Снять трассы 5–10 «горячих» сценариев (создание сделки, добавление товара, резерв).
2. Собрать топ N медленных SQL и провести индексацию.
3. Внедрить нагрузочное тестирование (k6/JMeter) для ключевых API.

## Следующие шаги
- 🟨 Ответственный: Backend + QA
- ⏱ Срок: 3 недели
- 📌 Контроль: отчёт по профилированию + план оптимизации.
