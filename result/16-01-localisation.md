# 16.1. Локали

## Результат
- **Статус:** ⚠️ Требует улучшения
- **Severity:** S3
- **Краткое заключение:** часть строк захардкожена в коде; не все сообщения вынесены в `lang/*`.

## Наблюдение
- В контроллерах и helper'ах встречаются строки на русском/английском (например, сообщения в `helpers.php`, `TestController`).
- Файлы `lang/ru.json`, `lang/en` присутствуют, но покрывают только часть функционала.

## Доказательства
- `app/Helpers/helpers.php` — строки «Да», «Нет», «валюта».
- `app/Http/Controllers/Api/V1/TestController.php` — русские фразы в коде.

## Риски
- Сложно поддерживать многоязычность/локализацию.
- Неконсистентные тексты в UI/письмах.

## Рекомендации
1. Вынести все пользовательские строки в `lang/*`.
2. Использовать `__('ключ')` вместо прямых строк.
3. Проверить email-шаблоны и UI на смешение языков.

## Следующие шаги
- 🟨 Ответственный: Backend + Frontend
- ⏱ Срок: 3 недели
- 📌 Контроль: линтер или PR check, запрещающий новые строки вне словарей.
