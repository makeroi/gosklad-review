# 21.1. Целевые метрики

## Результат
- **Статус:** ⚠️ Требует определения
- **Severity:** S2
- **Краткое заключение:** целевые метрики не зафиксированы; фактические показатели не измеряются.

## Наблюдение
- На звонке указано: «ожидания складов/остатков более 10 сек».
- Нет Dashboard’ов и SLA/SLO для API.

## Доказательства
- Отсутствие мониторинга (см. `09.3`)

## Риски
- Невозможно понять, достигнут ли приемлемый уровень производительности.
- Нет базы для договорённостей с бизнесом по SLA.

## Рекомендации
1. Зафиксировать целевые значения (API P95 < 800 мс, UI < 2 сек, успех очередей > 99%).
2. Настроить сбор и визуализацию метрик (Prometheus + Grafana).
3. Обновлять показатели еженедельно.

- ⏱ Срок: 2 недели
