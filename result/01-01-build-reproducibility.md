# 1.1. Версии и воспроизводимость сборки

## Результат
- **Статус:** ❌ Не пройдено
- **Severity:** S1
- **Краткое заключение:** версии ключевых зависимостей не зафиксированы должным образом; сборка зависит от глобального окружения.

## Наблюдение
- `composer.json` содержит `minimum-stability: dev`, что позволяет вытягивать нестабильные версии пакетов.
- Отсутствуют Dockerfile/Makefile и инструкции по воспроизводимому запуску.
- README не описывает порядок установки расширений PHP.

## Доказательства
- `composer.json` — строки 74-75 (minimum-stability)
- `README.md` — отсутствует раздел «Установка/запуск»

## Риски
- Разные разработчики и сервера могут устанавливать несовместимые версии библиотек.
- Высокая вероятность непредсказуемых падений после `composer install`.

## Рекомендации
1. Убрать `minimum-stability: dev`, оставить `prefer-stable: true`.
2. Подготовить Dockerfile или инструкции по настройке окружения (PHP extensions, Composer version, Node).
3. Зафиксировать версии в `package.json` и `composer.lock`, добавить make-таргет `make bootstrap`.
