# 17.2. Статический анализ

## Результат
- **Статус:** ❌ Не пройдено
- **Severity:** S2
- **Краткое заключение:** статический анализ (PHPStan/Larastan, Psalm) не настроен; в CI отсутствуют проверки качества кода.

## Наблюдение
- В `composer.json` нет dev-зависимостей Tatistan/Psalm.
- Нет конфигураций `.phpstan.neon`, `.psalm.xml`.
- CI/CD отсутствует → статический анализ не запускается автоматически.

## Доказательства
- `composer.json` — отсутствуют инструменты анализа
- Отсутствие CI конфигов

## Риски
- Невозможность обнаружить типовые ошибки и dead code до продакшна.
- Повышенный риск регрессий при рефакторинге.

## Рекомендации
1. Добавить `larastan/larastan` и/или `vimeo/psalm` в `require-dev`.
2. Настроить конфигурацию и поднять уровень анализа до 5-6.
3. Запускать анализ в CI для каждого PR.
